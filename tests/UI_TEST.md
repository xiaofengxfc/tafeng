# UI 测试指南 — Mobile-First 样式

## 环境

```bash
npm run dev        # 前端热更新 http://localhost:5173
npm run worker:dev # Worker http://127.0.0.1:8787
```

## 测试断点

打开浏览器 F12 → 设备模拟，拖拽宽度测试以下尺寸：

| 断点 | 目标设备 | 预期布局 |
|---|---|---|
| **320px** | iPhone SE | 单列，面板全折叠，终端 10px 字体 |
| **390px** | iPhone 14/15 | 单列，面板全折叠，终端 12px 字体 |
| **520px** | 折叠屏/小平板 | 单列过渡，面板开始可折叠 |
| **768px** | iPad 竖屏 | 3 列网格：220px \| 1fr \| 240px，面板展开 |
| **1120px** | 笔记本 | 3 列网格：230px \| 1fr \| 250px |
| **1440px** | 桌面 | 3 列网格：250px \| 1fr \| 280px |

## 测试清单

### 1. 登录页
- [ ] 深色/浅色主题切换
- [ ] 手机端布局自适应
- [ ] 输入框 focus 蓝色发光

### 2. 连接面板（左侧）
- [ ] 连接列表显示
- [ ] 展开/折叠操作按钮
- [ ] 点击「+」展开添加表单
- [ ] 编辑 VPS → 表单自动展开并填充数据
- [ ] 表单字段可正常输入

### 3. SSH 终端
- [ ] 连接 SSH 后终端显示
- [ ] 终端宽度不超过屏幕（无横向滚动条）
- [ ] fit 自适应不报错

### 4. 文件管理器
- [ ] 点击「▸ 文件管理」展开
- [ ] 文件列表可滚动
- [ ] 点击文件 → 编辑器显示内容
- [ ] 编辑器可编辑（contentEditable）
- [ ] 编辑区不溢出屏幕

### 5. 命令历史
- [ ] 点击展开
- [ ] 命令列表显示
- [ ] 搜索过滤正常
- [ ] 列表可滚动

### 6. 实时状态
- [ ] 手机端在 center-stack 内可折叠
- [ ] 桌面端在右侧固定显示
- [ ] CPU/内存/磁盘进度条正常
- [ ] 进程列表正常显示

### 7. 浅色主题
- [ ] 所有面板背景正确
- [ ] 文字可读
- [ ] 分割线可见

### 8. 触屏（真实手机）
- [ ] 按钮 ≥ 44px 触控区域
- [ ] 滚动流畅
- [ ] 输入框 focus 时自动弹出键盘
